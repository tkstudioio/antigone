# Data layer (cross-cutting)

Persistence on **PostgreSQL** via Prisma Client (PostgreSQL provider, `@prisma/adapter-pg`).

## Entities

`prisma/schema.prisma`. Domain entities: `Account`, `Challenge`, `Product`, `Key` (license key for
sale), `Favorite`, `Chat`, `Message`, `MessageKey`, `Escrow`, `Order`, `Review`. All
column/table names are snake_case, preserved via `@map`/`@@map`.

### Key relations

- A `Key` is linked to its escrow **indirectly** through its order:
  `Key.orderId → Order.escrowAddress → Escrow`. There is **no** direct key↔escrow FK.
- `Order.escrow` is a navigable direct Prisma 1-1 relation toward `Escrow` (virtual field, **no new column**): `Order.escrowAddress @unique → Escrow.address @id`. It allows `include: { escrow: { select: { status: true } } }` without a separate query. Used by `queryDisputes()` to expose `escrowStatus` in the dispute list.
- `Key.code` is encrypted at rest, `Key.code_hash` is an HMAC for dedup — see
  [crypto/at-rest.md](./crypto/at-rest.md).
- The text of `Message`s is E2E-encrypted, with one `MessageKey` per recipient — see
  [crypto/messaging.md](./crypto/messaging.md).
- `Escrow.arbiterPubkey` references the admin `Account` (created via `upsert` at checkout).

## Enums

`src/db/enums.ts` — TS `as const` arrays for the status fields (`chatStatusValues`,
`orderStatusValues`, `escrowStatusValues`, `reviewerRoleValues`). **Import from here**, not from the types
generated by Prisma. The meaning of the order/escrow statuses is in [lifecycle.md](./lifecycle.md).

## Client and adapter

- `src/db/index.ts` — `db` is a `PrismaClient` singleton backed by `PrismaPg` (from
  `@prisma/adapter-pg`). `DATABASE_URL` is **required** (throws at import). HMR-safe via `globalThis`
  in development.
- `prisma.config.ts` — Prisma 7 config file; sets `DATABASE_URL` from env for migrations and the adapter.

## Migrations

In `prisma/migrations/`. The initial migration `0001_init` is baselined against the existing DB.
Commands: `yarn db:migrate` (deploy), `yarn db:migrate:dev` (create+apply in dev), `yarn db:generate`
(regenerate the typed client).

## Case-insensitive search

Use `mode: "insensitive"` on Prisma `contains`, or `lower()` in `$queryRaw` for complex
aggregation queries.
